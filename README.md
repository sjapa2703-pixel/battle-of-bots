## Что где лежит

* `/backend` — наш сервак на Node.js (Express) + база данных MongoDB.
* `/frontend` — сама приложуха (React SPA).

## Как быстро запустить

1. Открываешь терминал прям в главной папке (`shop.nhrl.io`).
2. Пишешь одну команду, чтобы подтянуть все пакеты:
   ```bash
   npm run install:all
   ```
3. Стартуешь бэк и фронт одновременно:
   ```bash
   npm run dev
   ```

*Всё, сайт откроется на `http://localhost:5173`, а апишка будет висеть на `http://localhost:5000`.*

## Настройка ключей (Обязательно)

Перед деплоем или полноценным использованием необходимо настроить ключи базы данных:

1. Заходишь в файл `backend/.env`.
2. В `MONGO_URI` вставляешь реальную ссылку на свою базу (например, от MongoDB Atlas).
3. В `JWT_SECRET` вписываешь какую-нибудь рандомную кракозябру (пароль), чтобы данные юзеров не слили.

>Кстати про оффлайн: Если тебе влом ставить Монгу или нет инета, сервак запустится в "оффлайн-режиме". База работать не будет, но на фронте подтянутся фейковые данные, чтобы можно было просто покликать и заценить дизайн.

## Верификация почты

я тут прикрутил подтверждение по почте при регистрации, чтобы всякие боты не спамили аккаунтами. 
Короче, как это настроить, чтобы письма реально улетали юзерам:

1. Заходишь все в тот же `backend/.env`.
2. Тебе нужно добавить настройки SMTP-сервера (подойдет любой: Gmail, Яндекс, Mail.ru). Допиши туда вот эти строчки:
   ```env
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=твой_email@gmail.com
   SMTP_PASS=твой_пароль_приложения
   ```

 Если юзаешь Gmail твой обычный пароль от почты не прокатит изза безопасности. Надо зайти в настройки гугл аккаунта(в раздел безопасность) и создать "Пароль приложений" (App Password). Вот именно эту сгенерированную дичь и вставляй в `SMTP_PASS`

А если тебе влом сейчас с этим возиться или ты просто хочешь затестить регистрацию локально просто забей и ничего не пиши про SMTP в `.env`. 
Сервак умный, он сам создаст фейковый тестовый ящик(ethereal email) и прямо в консоли бэкенда (в терминале) при регистрации выведет ссылку типа: `Verification email preview URL: ...`. Просто кликаешь по ней в терминале у тебя откроется фейковое письмо, и там жмешь ссылку верификации

## Как задеплоить

### Заливаем бэк (на Render, Railway или Heroku)

1. В настройках хостинга пропиши переменные (Environment Variables):
* `PORT=5000`
* `MONGO_URI=твоя_ссылка_на_базу`
* `JWT_SECRET=твой_секретный_код`

Команда для старта сервера:
```bash
npm start
```

### 2. Деплой Фронтенда (Vercel, Netlify)

1. При импорте проекта на хостинг укажите корневую папку (Root Directory) как `frontend`.
2. Команда для сборки (Build Command):
   ```bash
   npm run build
   ```
3. **Супер важно:** Перед заливкой не забудь поменять в коде все локальные ссылки (`/api/...` или `localhost:5000`) на реальный адрес твоего бэкенда в инете, иначе ничего не законнектится!
